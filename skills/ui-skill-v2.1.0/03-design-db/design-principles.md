# 设计原则 (Design Principles)

## 创意北极星

本设计系统面向 **AI 平台型企业级 Web 应用**，追求"克制的专业感"。

> **核心哲学：信息第一，装饰为零。**

- 通过蓝色主调传递可信赖感
- 通过无 Card 包裹的扁平布局减少视觉噪音
- 通过宽松的间距系统保持内容的呼吸感
- 不追求视觉冲击，让每个组件在正确位置以最准确形式传递信息

## 原则优先级

**一致性 > 可读性 > 美观性**

---

## 万擎顶层设计原则

三大核心维度：**高效**（展示 / 操作）、**愉悦**（感受 / 情绪）、**直觉**（认知 / 预期）

---

## 高效 — 展示 / 操作

### 清晰度
视觉元素展示完整清晰，文字可读性强，重点突出，信息层级及从属关系清晰，控件交互及界面跳转逻辑清晰

- 核心原则01：内容展示完整清晰
- 核心原则02：布局逻辑清晰
- 核心原则03：信息交互逻辑清晰

### 一致性
同平台产品、同业务功能模块有一致性，相似属性的视觉交互体验以及文本一致

- 核心原则04：界面保持一致性

### 可见性
功能及界面元素易于发现，控件的可操作性易于发现，明确当前操作状态及进度，并提供即时、有效的操作反馈

- 核心原则05：功能及信息保持可见性
- 核心原则06：系统状态保持可见性
- 核心原则07：用户的操作产生必要反馈

### 操作便捷性
核心操作突出，常用功能放在最显眼、顺手的位置，精简操作步骤，减少用户操作阻碍和负荷，提供符合用户下意识的操作

- 核心原则08：操作展示清晰
- 核心原则09：减少用户操作
- 核心原则10：操作方便易找

### 防错及容错性
提前预防用户犯错，主动告知用户错误类型以及如何恢复，帮助用户纠错并扶慰用户

- 核心原则11：预防并容许用户犯错

---

## 愉悦 — 感受 / 情绪

### 审美性
界面元素美观，留白合理，布局规整，重复排列有韵律，用色符合逻辑

- 核心原则12：美服务与功能

### 帮助性
提供必要的新功能介绍和操作说明，提供帮助中心或客服

- 核心原则13：为用户主动提供帮助

### 情感化
人性化表达，避免使用机器语言和用户沟通

- 核心原则14：和用户进行情感化交流

### 智能化
自动化智能操作，为用户进行自动整理，根据场景进行功能和内容的智能适配

- 核心原则15：提供智能化操作

---

## 直觉 — 认知 / 预期

### 易操作性
界面符合用户认知习惯，视觉含义易于理解，寓意手法符合直觉和经验，文本描述清晰易理解，操作方式易理解

- 核心原则16：用户能理解界面信息

### 易学性
操作方法易于学习和掌握，尽量复用已有操作习惯，降低认知负担，避免信息过载

- 核心原则17：降低用户的认知负荷

### 可预测性
操作目的和结果可预期，操作结果和用户预期保持一致，避免误操作，预判用户下一步操作，提前为其提供直接操作

- 核心原则18：用户的操作需要达到预期效果

---

## 五条核心设计原则

### 1. 扁平化布局（Flat Layout）
- Table 直接平铺，**不使用 Card 包裹**
- 内容区通过背景色差分层，而非 box-shadow
- 禁止在内容区使用卡片的阴影效果

### 2. 克制的色彩（Restrained Color）
- 颜色仅用于传递语义信息（主操作、状态、错误）
- 正文区域以中性色 `#252626 / #575859 / #898A8C` 为主
- 避免使用装饰性色块、渐变背景（PageHeader 渐变除外）

### 3. 清晰的层级（Clear Hierarchy）
- TopNav → Sider → PageHeader → Content 的视觉层级清晰
- 字号层级：18px（页标题）→ 16px（区块标题）→ 14px（正文）→ 12px（辅助）
- 操作按钮优先级：Primary > Default > Link

### 4. 一致的反馈（Consistent Feedback）
- 加载状态：Table `loading` prop
- 成功操作：`message.success()`
- 错误状态：`message.error()` 或表单验证提示
- 空状态：统一使用 `–`（颜色 `#BBBDBF`），禁止留空

### 5. 框架统一（Unified Framework）
- **所有页面必须通过 `AppLayout` 提供框架**（TopNav + Sider）
- 页面组件不得自行实现导航栏或侧边栏
- AppLayout 文件位于 `02-templates/AppLayout/index.tsx`

## 禁止行为

| 禁止项 | 原因 |
| :--- | :--- |
| 页面自行实现 TopNav / Sider | 破坏框架一致性 |
| Table 使用 Card 包裹 | 破坏扁平化风格 |
| Table `bordered={true}` | 竖向分割线造成视觉噪音 |
| 硬编码颜色值（如 `#326BFB`）| 绕过 Token 系统，难以维护 |
| 空值留空不显示 `–` | 用户无法判断是空还是加载中 |
| `destroyOnClose`（Modal）| 应使用 `destroyOnHidden`（antd v5） |
| 内容区 Card 使用 box-shadow | 应通过背景色差分层 |
| 详情页用 Card 分组 | 应使用区块标题 + Divider + Descriptions |
