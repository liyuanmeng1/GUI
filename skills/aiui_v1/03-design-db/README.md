## 此目录应包含的文件

### `color.md`（必须）

颜色规范，至少包含：
- 品牌主色（Primary）色值
- 功能色（成功/警告/错误）色值
- 中性色（文字层级、边框、背景）色值
- 页面结构色（页面背景、卡片背景）

没有设计稿也没关系，告诉 AI 主色色值，AI 会自动补全完整颜色系统。

---

### `typography.md`（必须）

字体规范，至少包含：
- 全局字体栈
- 字号层级（页面标题/区块标题/正文/小字）
- 文字颜色层级

没有规范 → 告诉 AI「默认」，AI 会使用通用 B 端规范。

---

### `copywriting.md`（必须）

文案规范，至少包含：
- 产品语气风格（严肃专业/友好轻松/技术简洁）
- 按钮文案统一规范（新建/编辑/删除/确认/取消）
- 资源状态词统一（运行中/已停止/排队中等）
- 空值显示规范（统一用什么符号）

---

### `design-principles.md`（必须）

设计原则，至少包含：
- 交互反馈原则（成功/失败/加载如何反馈）
- 危险操作保护（删除确认等）
- 禁止行为清单（什么样式不允许用）

没有规范 → 告诉 AI「默认」，AI 会使用通用设计原则。

---

### `component-spec.md`（可选但推荐）

组件规范，记录以下内容（防止 AI 自造轮子）：
- 组件库中有哪些基础组件（表格、表单、对话框等）
- 每个组件的标准用法和禁止用法
- 特殊注意事项（如分页器语言包、Modal 关闭行为等）

---

### 其他扩展文件（可选）

| 文件名 | 内容 |
|--------|------|
| `layout-patterns.md` | 页面布局模式（PageHeader/Toolbar/详情分区等） |
| `interaction-rules.md` | 交互规则（加载/空态/错误状态处理） |
| `state-design.md` | 资源状态设计（状态机/状态色语义） |
| `chart-spec.md` | 图表规范（图表类型/颜色/交互） |
| `anti-patterns.md` | 明确的禁止行为清单 |
