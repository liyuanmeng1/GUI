# aiui_v1 开箱配置指南

> 本文档帮助业务团队快速了解如何配置 `aiui_v1` skill，适合发给接入团队的同学阅读。

---

## 什么是 aiui_v1？

`aiui_v1` 是一个通用企业级 UI 生成 Skill。配置完成后，你可以直接告诉 AI：
- 「帮我生成一个任务列表页」
- 「根据这个 PRD 生成三个页面的代码」
- 「新建表单页，有基础信息和高级配置两个区块」

AI 会自动按照你们的技术栈、组件库、设计规范生成符合标准的代码。

---

## 配置前的准备

在开始前，准备好以下材料（**有什么拿什么，没有可以跳过**）：

### 第一层：基础层（必须）
- [ ] 项目的 `package.json`（或者告诉我用什么组件库）
- [ ] 现有的主题配置代码（如 antd ConfigProvider、CSS 变量）
- [ ] 品牌主色色值（如 `#165DFF`）

### 第二层：模板层（建议）
- [ ] 现有项目中的列表页代码（随便哪个都行）
- [ ] 现有项目中的详情页代码
- [ ] 现有项目中的表单页代码
- [ ] 全局框架（TopNav + Sider）代码

### 第三层：设计规范层（建议）
- [ ] 设计规范文档截图或链接
- [ ] 或者只需要：主色色值 + 产品语气（严肃/轻松）

---

## 如何开始配置

直接告诉 AI：

```
我要配置 aiui_v1 skill
```

AI 会一步步引导你完成所有配置，**每完成一项就写入文件，可以随时中断**。

---

## 配置完成后如何使用

配置完成后，重启 VS Code（或等待约 30 秒），然后直接发需求即可：

| 你说的话 | AI 做什么 |
|---------|---------|
| 「帮我生成一个任务列表页，包含任务名、状态、创建时间三列」 | 基于你的 ListPage 模板生成业务代码 |
| 「根据这个 PRD 生成代码」（附上 PRD） | 分析页面类型，读取对应模板，生成完整页面代码 |
| 「新建表单页，字段如下……」 | 基于你的 FormPage 模板生成表单代码 |
| 「这段代码有什么问题？」 | 对照你们的设计规范和组件规范做 Review |

## 常见问题

**Q：我们没有设计规范文档，可以配置吗？**

可以！只需要告诉 AI 主色色值 + 产品语气风格，AI 会自动生成一套完整的规范文档。

**Q：我们团队有私有组件库，可以用吗？**

可以！在配置时告诉 AI 组件库的包名和安装方式即可。

**Q：配置一半想休息，可以中断吗？**

可以，每完成一项 AI 都会立即写入文件。下次继续时告诉 AI「继续配置 aiui_v1」。

**Q：配置完成后想修改某一项，怎么办？**

直接告诉 AI 你想修改哪一层的内容，AI 会引导你重新填写并更新对应文件。

**Q：我想把我们现有项目的页面作为模板，可以吗？**

可以！把现有页面的代码直接发给 AI，AI 会帮你整理成规范的模板格式写入。

---

## 注意事项

1. **`02-templates/` 中的文件是只读模板**，请勿直接修改。开发业务页时 AI 会自动复制模板到 `src/pages/` 下再修改。

2. **配置完成后等待约 30 秒**让 CodeFlicker 检测到新文件，或重启 VS Code 立即生效。

3. **越详细越好**：发给 AI 的代码或设计稿越详细，生成的代码越贴近你们的实际规范。
