各自优势

🔷 A2UI — Google

  • 跨平台原生渲染（Web / iOS / Android / Flutter）
  • 协议级标准，与框架、传输层完全解耦
  • 邻接表 + 多消息类型，LLM 流式生成友好
  • 数据与结构分离，服务端独立推送 DataModel
  • 白名单 Catalog 作为安全边界（JSON Schema 标准）
  • 标准化 userAction 交互闭环，Agent 感知用户操作

🟢 json-render — Vercel Labs

  • React 专用，快速落地，上手成本低
  • 嵌套树结构 + Zod 类型验证，开发体验好
  • createSpecStreamCompiler 处理不完整 JSON 流
  • 客户端本地 State，$bindState 双向绑定，无需服务端参与
  • catalog.prompt() 自动生成 LLM System Prompt
  • 轻量灵活，无需完整协议栈

场景选型建议

🌐 Web 端快速验证

React 项目，快速跑通 AI 生成 UI 流程，无跨平台需求

推荐 json-render

📱 跨平台 Agent App

同一套 AI UI 需跑在 Web + iOS + Android，安全性要求高

推荐 A2UI

🏗️ 长期规模化系统

先用 json-render 验证，按 A2UI 标准设计 Catalog，保留迁移能力

A2UI Catalog + json-render Renderer

🔗 多 Agent 协作系统

Agent 运行在远端，UI 需实时响应，搭配 AG-UI 传输层

推荐 A2UI + AG-UI
